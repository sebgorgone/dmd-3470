The SASS Syntax

.sass .scss

.sass

@use "sass:list"

@use "sass:color"

$font-stack: Helvetica, Arial

$primary-color: #333

body

$font-stack: list.append($font-stack, sans-serif)

font: $font-stack

a

color: $primary-color

&:hover

color: color.scale($primary-color, $lightness: 20%)

.scss

@use "sass:list";

@use "sass:color";

$font-stack: Helvetica, Arial;

$primary-color: #333;

body {

$font-stack: list.append($font-stack, sans-serif);

font: $font-stack;

}

a {

color: $primary-color;

&:hover {

color: color.scale($primary-color, $lightness: 20%);

}

}

.css

body {

font: Helvetica, Arial, sans-serif;

}

a {

color: #333;

}

a:hover {

color: rgb(91.8, 91.8, 91.8);

}

Sass contains two different syntaxes where the defining difference between the two is the use of curly braces vs the use of indentation to denote code structure


The .sass syntax exists as a mode of backwards compatability as it was the orginal language for the sass compiler


It is less common as the .scss syntax allows any valid css code to be compiled as well as sass code
