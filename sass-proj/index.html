Sass contains two different syntaxes where the defining difference between the two is the use of curly braces vs the use of indentation to denote code structure


The .sass syntax exists as a mode of backwards compatability as it was the orginal language for the sass compiler


It is less common as the .scss syntax allows any valid css code to be compiled as well as sass code

SASS Compiler

.scss => .css

The sass preprocessor can be installed as a binary onto virtually any machine via the github or through a number of package managers such as npm, brew, pacman, etc..


The binary can be called to do a number of things most commonly to watch a .sass or .scss and output the compiled css into a specified file


Alternatively this action can be handled by a vscode extension

bash

> npm install -g sass

> brew install sass

> pacman -S sass

bash

# single use command

# compile a sass file to an output css file


> sass input.scss output.css

bash

# begin foreground preocess to

# compile a sass file to

# an output css file when

# changes are made


> sass --watch input.scss:output.css

bash

# run the sass compiler

# as a background process


> sass --watch input.scss:output.css &


# NOTE** you will have to manually end

# this process by whatever

# means your operating system expects

The sass errors and debug statements etc will get printed to the terminal after each compile

Errors will also populate within your html if the sass code that you right cannot be compiled

Its important to note that like a handful of other css feautures variables are not exclusive to sass


However sass handles its variables in a different way than css variables alone


Since sass compiles down to css it has the oppurtunity to give variables muteable states before compile time making the variables imperitive rather than declaritive


The examples below demonstrate this behavior

Mixins

@mixin @include

Mixins are reusable blocks of CSS that can be included in other rules. They allow you to define styles that can be reused throughout your stylesheet.


Mixins can take arguments, making them flexible. You define a mixin with @mixin name($arg) { ... } and include it with @include name(value).


Mixins find the majority of their use being reused as part of your sass documents and creating a more modular structure for your styles

.scss

@mixin flex-center($direction: row) {

display: flex;

justify-content: center;

align-items: center;

flex-direction: $direction;

}

.container {

@include flex-center();

}

.vertical {

@include flex-center(column);

}

Built-ins

Sass Functions

The lighten function increases the lightness of a color by a specified percentage, making it appear brighter.

.scss

@use "sass:color";

$base-color: #333;

$light-color: color.lighten($base-color, 20%);

.element {

background-color: $light-color;

}

.css

.element {

background-color: #666666;

}

The mix function blends two colors together based on a weight percentage, creating a new color.

.scss

@use "sass:color";

$color1: red;

$color2: blue;

$mixed-color: color.mix($color1, $color2, 50%);

.element {

color: $mixed-color;

}

.css

.element {

color: #800080;

}

The round function rounds a number to the nearest integer, useful for precise calculations.

.scss

@use "sass:math";

$value: 3.7;

$rounded: math.round($value);

.element {

width: $roundedpx;

}

.css

.element {

width: 4px;

}
